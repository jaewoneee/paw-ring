# 사용자 설정

## 개요
- 프로필 관리, 알림 설정, 앱 정보 등 유저 개인 설정 기능

## 목적
- 유저 프로필(닉네임, 이미지) 관리
- 알림 등 앱 환경 커스터마이징
- 이용약관, 개인정보 처리방침 등 법적 고지 제공

## 기술 스택
- **프론트엔드**: Expo (React Native) + TypeScript
- **DB**: Cloud Firestore
- **이미지 저장**: Firebase Storage
- **인증**: Firebase Auth (회원 탈퇴, 비밀번호 변경)

## 사용자 시나리오

### 시나리오 1: 프로필 수정
1. 사용자가 설정 탭에 진입한다
2. 프로필 영역(사진, 닉네임)을 탭한다
3. 프로필 수정 화면에서 닉네임 변경, 프로필 사진 변경/삭제
4. "저장"을 누르면 Firestore 유저 문서 업데이트

### 시나리오 2: 알림 설정
1. 설정 화면에서 "알림" 메뉴를 탭한다
2. 전체 알림 ON/OFF 토글
3. OFF로 변경하면 모든 푸시 알림 중지

### 시나리오 3: 로그아웃
1. 설정 화면 하단의 "로그아웃"을 탭한다
2. 확인 다이얼로그 표시
3. 확인 시 Firebase Auth signOut + 로그인 화면으로 이동

### 시나리오 4: 회원 탈퇴
1. 설정 화면 하단의 "회원 탈퇴"를 탭한다
2. 경고 다이얼로그: "모든 데이터가 삭제되며 복구할 수 없습니다"
3. 비밀번호 재입력 (보안 확인)
4. 확인 시:
   - Firestore 유저 데이터 삭제 (반려동물, 스케줄, 공유 등)
   - Firebase Storage 이미지 삭제
   - Firebase Auth 계정 삭제
5. 로그인 화면으로 이동

## 아키텍처 설계

### Frontend

```
app/
├── (tabs)/
│   └── settings.tsx              # 설정 메인 화면
├── settings/
│   ├── profile.tsx               # 프로필 수정
│   ├── change-password.tsx       # 비밀번호 변경
│   └── terms.tsx                 # 이용약관 WebView

services/
├── user.ts                       # 유저 프로필 CRUD
├── account.ts                    # 계정 관리 (탈퇴, 비밀번호 변경)
```

## 데이터 모델

기존 `users/{uid}` 문서 활용 (login.md 참고)

```typescript
interface UserProfile {
  uid: string;
  email: string;
  nickname: string;
  profileImage: string | null;
  provider: 'email' | 'google';
  emailVerified: boolean;
  notificationEnabled: boolean;     // 전체 알림 설정
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

## 예외 처리

| 상황 | 대응 |
|------|------|
| 닉네임 미입력 | "닉네임을 입력해주세요" |
| 닉네임 길이 초과 (>20자) | "닉네임은 20자 이내로 입력해주세요" |
| 프로필 이미지 용량 초과 | "사진 크기는 5MB 이하만 가능합니다" |
| 회원 탈퇴 시 비밀번호 불일치 | "비밀번호가 올바르지 않습니다" |
| 회원 탈퇴 시 재인증 필요 | `reauthenticateWithCredential` 후 삭제 |
| 구글 로그인 유저 비밀번호 변경 시도 | "구글 로그인 계정은 비밀번호를 변경할 수 없습니다" |
