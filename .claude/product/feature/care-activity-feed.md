# 돌봄 활동 피드 (Care Activity Feed)

## 개요

- 반려동물별 돌봄 활동을 타임라인 형태로 보여주는 기능
- 혼자 사용 시: 내 돌봄 히스토리 확인
- 공유 사용 시: 공동 돌봄 현황판 (누가 언제 뭘 했는지)

## 목적

- 돌봄 기록을 시간순으로 한눈에 파악
- 공유 멤버 간 별도 연락 없이 현황 공유
- 병원 방문 시 최근 돌봄 패턴 참고 자료로 활용

## 사용자 시나리오

### 시나리오 1: 혼자 사용 — 내 돌봄 기록 확인

1. 사용자가 홈 화면 또는 캘린더에서 활동 피드 탭/섹션에 진입한다
2. 최근 완료한 스케줄이 시간순 타임라인으로 표시된다
3. 각 항목에는 카테고리 색상, 스케줄 제목, 완료 시각이 표시된다
4. 날짜 구분선으로 일별 그룹핑된다

### 시나리오 2: 공유 사용 — 공동 돌봄 현황

1. 공유 캘린더의 활동 피드에 진입한다
2. 모든 공유 멤버의 완료 활동이 함께 표시된다
3. 각 항목에 "완료한 사람"이 아바타/닉네임으로 표시된다
4. "Jay님이 산책을 완료했어요 (14:30)" 형태

### 시나리오 3: 피드에서 상세 확인

1. 피드 항목을 탭하면 해당 스케줄 상세로 이동한다

## 주요 기능 요구사항

### 피드 표시

- [x] 반려동물별 활동 피드 화면
- [x] 완료된 스케줄을 시간 역순(최신순)으로 표시
- [x] 날짜별 그룹핑 (오늘 / 어제 / 날짜)
- [x] 카테고리 색상 표시
- [x] 완료 시각 표시
- [x] 완료자 정보 표시 (공유 캘린더인 경우)
- [ ] 무한 스크롤 또는 페이지네이션 (과거 기록 로드)

### 진입점

- [x] 홈 화면 — "최근 활동" 섹션 (최근 5개 미리보기)
- [ ] 캘린더 화면 — 피드 탭 또는 별도 진입점

### 필터

- [ ] 카테고리별 필터 (산책만 / 약만 등)
- [ ] 기간 필터 (오늘 / 이번 주 / 이번 달)

## 화면 구성

### 피드 아이템

```
┌─────────────────────────────────────┐
│  ── 오늘 ──────────────────────────  │
│                                     │
│  🟢 산책       Jay       14:30     │
│  🔵 약 투여    Jay       09:00     │
│                                     │
│  ── 어제 ──────────────────────────  │
│                                     │
│  🟢 산책       민지      18:00     │
│  🟠 식사       Jay       12:30     │
│  🔵 약 투여    Jay       09:00     │
│                                     │
└─────────────────────────────────────┘
```

- 좌측: 카테고리 색상 도트 + 스케줄 제목
- 중앙: 완료자 (공유 시)
- 우측: 완료 시각

### 혼자 사용 시

- 완료자 칼럼 숨김 (본인만 있으므로 불필요)
- 나머지 레이아웃은 동일

## 데이터

### 기존 테이블 활용

`schedule_completions` 테이블에 이미 필요한 데이터가 있음:

| 필드 | 용도 |
|------|------|
| `schedule_id` | 어떤 스케줄을 완료했는지 → JOIN으로 제목/카테고리 조회 |
| `completion_date` | 날짜별 그룹핑 |
| `completed_by` | 누가 완료했는지 → JOIN으로 닉네임/아바타 조회 |
| `completed_at` | 완료 시각 표시 |

### 쿼리 예시

```sql
SELECT
  sc.completed_at,
  sc.completion_date,
  s.title,
  s.category_id,
  u.nickname,
  u.profile_image
FROM schedule_completions sc
JOIN schedules s ON sc.schedule_id = s.id
JOIN users u ON sc.completed_by = u.id
WHERE s.pet_id = :petId
ORDER BY sc.completed_at DESC
LIMIT 20 OFFSET :offset;
```

### 추가 테이블/필드 변경

- 없음. 기존 스키마로 구현 가능.

## 구현 우선순위

### Phase 1: 기본 피드

- 반려동물별 활동 타임라인 화면
- 날짜별 그룹핑 + 시간순 정렬
- 스케줄 상세 이동

### Phase 2: 홈 화면 미리보기

- 홈 화면에 "최근 활동" 섹션 추가 (최근 3~5개)

### Phase 3: 필터 + 페이지네이션

- 카테고리/기간 필터
- 무한 스크롤로 과거 기록 로드
