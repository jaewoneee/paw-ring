# 반려동물 관리

## 개요
- 반려동물 프로필을 등록/수정/삭제하고, 목록으로 조회하는 기능
- 반려동물 1마리 = 캘린더 1개의 기본 단위

## 목적
- 반려동물 정보를 체계적으로 관리
- 캘린더/스케줄의 기본 단위가 되는 반려동물 엔티티 생성
- 여러 반려동물을 키우는 유저를 위한 멀티 프로필 지원

## 기술 스택
- **프론트엔드**: Expo (React Native) + TypeScript
- **DB**: Cloud Firestore
- **이미지 저장**: Firebase Storage
- **상태 관리**: React Context 또는 Zustand

## 사용자 시나리오

### 시나리오 1: 반려동물 등록
1. 사용자가 메인 화면에서 "반려동물 추가" 버튼을 누른다
2. 반려동물 등록 화면이 표시된다
3. 필수 정보를 입력한다: 이름, 종류(강아지/고양이/기타)
4. 선택 정보를 입력한다: 품종, 생년월일, 성별, 몸무게, 프로필 사진
5. "등록하기" 버튼을 누른다
6. Firestore에 반려동물 문서가 생성된다
7. 해당 반려동물의 캘린더가 자동 생성된다
8. 메인 화면(반려동물 목록)으로 돌아간다

### 시나리오 2: 반려동물 목록 조회
1. 사용자가 메인 화면에 진입한다
2. 등록된 반려동물 목록이 카드 형태로 표시된다
3. 각 카드에 프로필 사진, 이름, 종류가 표시된다
4. 카드를 탭하면 해당 반려동물의 캘린더로 이동한다

### 시나리오 3: 반려동물 프로필 수정
1. 반려동물 카드의 설정 아이콘 또는 프로필 상세에서 "수정"을 누른다
2. 기존 정보가 채워진 수정 화면이 표시된다
3. 원하는 항목을 수정한다
4. "저장" 버튼을 누르면 Firestore 문서가 업데이트된다

### 시나리오 4: 반려동물 삭제
1. 프로필 수정 화면 하단의 "삭제하기"를 누른다
2. 확인 다이얼로그가 표시된다: "테오의 모든 스케줄이 삭제됩니다. 정말 삭제하시겠습니까?"
3. "삭제"를 선택하면 반려동물 + 연관 캘린더/스케줄이 삭제된다
4. 메인 화면으로 돌아간다

## 주요 기능 요구사항

### 반려동물 등록
- [ ] 필수 입력: 이름, 종류 (강아지/고양이/기타)
- [ ] 선택 입력: 품종, 생년월일, 성별(수컷/암컷/중성), 몸무게, 프로필 사진
- [ ] 프로필 사진 업로드 (카메라 촬영 / 갤러리 선택)
- [ ] 등록 시 캘린더 자동 생성

### 반려동물 조회
- [ ] 내 반려동물 목록 조회 (카드 UI)
- [ ] 공유받은 반려동물도 별도 섹션으로 표시

### 반려동물 수정
- [ ] 모든 필드 수정 가능
- [ ] 프로필 사진 변경/삭제

### 반려동물 삭제
- [ ] 삭제 확인 다이얼로그 (되돌릴 수 없음 경고)
- [ ] 연관 데이터 일괄 삭제 (캘린더, 스케줄, 공유 관계)

## 데이터 모델

### Firestore: `pets` 컬렉션
```typescript
// pets/{petId}
interface Pet {
  id: string;                     // 문서 ID (auto-generated)
  ownerId: string;                // 생성자 UID
  name: string;                   // 이름 (필수)
  type: 'dog' | 'cat' | 'other'; // 종류 (필수)
  breed: string | null;           // 품종
  birthDate: Timestamp | null;    // 생년월일
  gender: 'male' | 'female' | 'neutered' | null; // 성별
  weight: number | null;          // 몸무게 (kg)
  profileImage: string | null;    // 프로필 이미지 URL (Firebase Storage)
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

### Firestore 보안 규칙
```
match /pets/{petId} {
  // 소유자만 읽기/쓰기
  allow read, write: if request.auth != null && request.auth.uid == resource.data.ownerId;
  // 생성 시 ownerId가 본인인지 확인
  allow create: if request.auth != null && request.auth.uid == request.resource.data.ownerId;
  // 공유받은 유저도 읽기 가능 (sharing 기능에서 확장)
}
```

3. 반려동물 삭제 + 연관 데이터 정리
