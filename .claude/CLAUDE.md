# CLAUDE.md - 필수 규칙

## 사고 및 답변 규칙

1. **Think in English**: 사용자가 한국어로 질문하면, 먼저 영어로 번역하여 영어로 사고한다. 답변 시에는 한국어로 번역하여 응답한다.

## 코딩 규칙

- 날짜/시간 포맷팅 시 반드시 `@/utils/dayjs`의 공통 모듈(dayjs 인스턴스 및 포맷 함수)을 사용한다. `dayjs`를 직접 import하지 않는다.

## 기획 문서 작성 및 동기화

- 기획 문서 작성 시 `.claude/product/` 하위 폴더의 `README.md`를 참고하여 적절한 폴더에 작성한다.
  - `feature/` — 기능별 상세 기획 (화면, 데이터 모델, 체크리스트)
  - `design/` — 디자인 시스템 (컬러, 타이포그래피)
  - `decision/` — 기획 단계 설계 결정 (선택지 비교, 이유)
  - `history/` — 구현 중 방향 전환/구조 변경 이력
  - `improvement/` — 코드 품질·성능·UX 구조 개선 과제 추적
- 코드를 추가하거나 수정할 때, `.claude/product/` 디렉토리의 기획 문서와 연관이 있는지 확인한다.
- 연관이 있다면 해당 기획 문서도 함께 업데이트한다 (화면 구성, 데이터 모델, 기능 요구사항 체크리스트 등).
- 구현 완료된 항목은 `- [x]`로, 미구현 항목은 `- [ ]`로 표시한다.
- 기능 구현 완료 시 `.claude/product/feature/README.md`의 해당 항목 상태를 반드시 ✅로 업데이트한다.
