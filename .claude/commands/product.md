사용자가 설명하는 기능을 기획하고, .claude/product 폴더에 기능명.md로 정리해줘.

## 입력

$ARGUMENTS

## 프로세스

1. 사용자의 기능 설명을 분석한다.
2. 아래 템플릿에 맞춰 기획서를 작성한다.
3. `.claude/product/` 폴더가 없으면 생성한다.
4. `.claude/product/기능명.md` 파일로 저장한다.
   - 기능명은 영어, 간결하게 (예: login.md, profile.md)
5. 저장 완료 후 기획 요약을 보여준다.

## 기획서 템플릿

```markdown
# 기능명

## 개요
- 기능에 대한 한 줄 요약

## 목적
- 이 기능이 왜 필요한지

## 사용자 시나리오
1. 사용자가 ~한다
2. 시스템이 ~한다
3. ...

## 주요 기능 요구사항
- [ ] 요구사항 1
- [ ] 요구사항 2
- [ ] ...

## 화면 구성
- 필요한 화면/UI 요소 설명

## 데이터 모델
- 필요한 데이터 구조 정의

## 예외 처리
- 에러/예외 상황과 대응 방안

## 참고 사항
- 추가 고려 사항이나 제약 조건
```

## 규칙

- 한국어로 작성한다.
- 사용자가 설명하지 않은 부분은 합리적으로 추론하되, 추론한 내용은 _(추정)_ 으로 표시한다.
- 기획 내용에 대해 사용자에게 확인할 사항이 있으면 기획서 하단에 "확인 필요 사항" 섹션을 추가한다.
